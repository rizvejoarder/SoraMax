## üé® Screenshot Best Practices

### For GitHub Display

1. **Image Size**: 
   - Main screenshots: 1920x1080 or 1600x900
   - Feature highlights: 800x600 minimum
   - GitHub automatically scales images

2. **File Size**:
   - Optimize PNG files (use tools like TinyPNG)
   - Target: Under 500KB for fast loading
   - Use PNG-8 for simple screenshots
   - Use PNG-24 for gradient/photos

3. **Content**:
   - Show the entire application window
   - Include window title bar
   - Display meaningful data/prompts
   - Use default/professional settings

4. **Markdown Reference**:
   ```markdown
   ![Alt Text](screenshots/filename.png)
   ```

## üîß Taking Screenshots

### Windows
- **Full Window**: `Alt + PrtScn`
- **Snipping Tool**: Search for "Snip & Sketch"
- **Recommended**: Use Snipping Tool for clean captures

### macOS
- **Full Screen**: `Cmd + Shift + 3`
- **Selected Area**: `Cmd + Shift + 4`
- **Window Capture**: `Cmd + Shift + 4`, then `Space`

### Linux
- **GNOME**: `PrtScn` or Screenshot app
- **KDE**: Spectacle screenshot tool
